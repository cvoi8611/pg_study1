On-premise와 Off-premise

온 프레미스(On-premise)는 IT 서비스 운영을 위해, 직접적으로 인프라를 구축하는 방식

다시 말해, 온 프레미스라는 시스템을 구축하기 위해 하드웨어와 소프트웨어를 구입하여, 시스템 구성 상황에 맞게 환경을 구성하는 것을 말함

온 프레미스의 단점으로 이러한 인프라를 구축하기 위한 시간이 필요하며, 시스템을 구축하기 위한 물리적인 하드웨어 장비 구매 및 추가적으로 이를 관리, 운용, 유지보수하기 위한 비용이 요구됨

오프 프레미스(Off-premise)는 온 프레미스와 반대로 인터넷 네트워크에 연결된 서버팜이나 클라우드의 원격 실행환경

즉, 오프 프레미스는 인터넷 네트워크에 연결된 서버팜이나 클라우드의 원격 실행환경에서 사용되는 소프트웨어를 나타냄 오프 프레미스는 서드파티 공급업체가 인프라, 시설 및 관련 서비스를 제공하고 유지 관리하므로 조직은 서드파티 공급업체가 유지 관리하는 인프라를 사용
온 프레미스(On-premise) vs 오프 프레미스(Off-premise)
종류 온 프레미스(On-premise) 오프 프레미스(Off-premise)
관리 조직이 전체 인프라의 실행, 관리를 전적으로 책임짐 클라우드 서비스 공급업체가 책임짐
초기 비용 인프라 전체를 구축하는 방식으로, 일반적으로 대규모 자본 지출이 요구됨 따라서 상황이 변화하더라도 프로젝트를 도중에 포기, 변경이 힘들 수 있음 클라우스 서비스 공급업체에게 사용량 기반의 과금 모델을 제공하여 사용자는 구독 방식으로 인프라를 제공받을 수 있음
지속적 비용 기업이 에너지, 유지보수 및 관리 인력 비용을 지속해서 지출함, 매년 지출되는 비용으로 오프-프레미스 방식보다 약간 낮을 수 있음 구독 요금 또는 정액을 지불하기만 하면 되며, 시설 및 리소스 유지 관리와 관련된 비용도 이에 포함됨
소유권, 통제 조직이 인프라에 대한 완전 소유권을 가짐, 인프라 문제는 사내 IT 팀이 전적으로 책임지고 해결함 클라우드 서비스 공급업체와 조직은 집주인과 세입자와 같음, 클라우드 서비스 공급업체는 시설과 컴퓨팅 리소스를 소유하고, 계약에 따라 조직에게 인프라를 제공함
확장성 물리적으로 리소스가 존재하여 확장성에 제한이 있으며, 이 때문에 조직의 요구사항이 변경되면 리소스 부족 또는 리로스의 활용도가 낮아질 수 있음 클라우드 공급업체는 더 큰 리소스 확장성을 제공하므로 사용자는 수요에 따라 리소스를 확장하거나 축소하는것 까지 가능하여 필요한 리소스량을 거의 항상 사용 가능함

IDC

IDC : Internet Data Center

데이터 센터는 서버 컴퓨터와 네트워크 회선 등을 제공하는 건물 또는 시설을 말함

데이터 센터는 인터넷에서의 방대한 정보를 저장하고 웹 사이트에 표시하기 위해 수천, 수만대의 서버 컴퓨터를 한 장소에 모아 안정적으로 관리하기 위한 목적으로 건립하게 됨

여기서 통신업체의 데이터 센터는 인터넷 데이터 센터 (IDC), 클라우드 서비스를 위한 데이터 센터는 클라우드 데이터 센터(CDC)라고 한다.

IDC는 인터넷 연결의 핵심이 되는 서버를 한 데 모아 집중시켜야 할 떄, 설립하는 시설이며 다른 말로는 서버 호텔(Server Hotel)이라고 함

IaaS / Paas / SaaS

클라우딩 서비스 종류는 총 3가지로 나뉘어진다

  • Iaas (Infrastructure as a Service) : 개발사에 제공되는 물리적 자원을 가상화

    • IaaS는 기업이 준비해놓은 환경에서 우리가 OS를 선택하여 구동하는 가상 호스팅과 비슷함
    • ex - AWS의 EC2

  • Paas (Platform as a Service) : 개발사에 제공되는 플랫폼을 가상화

    • IaaS 형태의 가상화된 클라우드 위에 사용자가 원하는 서비스를 개발할 수 있도록 개발 환경(Platform)을 미리 구축해, 이를 서비스 형태로 제공하는 것이다.
    • ex - Lambda, Google App Engine

  • Saas (Software as a Service) : 고객에게 제공되는 소프트웨어를 가상화

    • 클라우드 서비스 형태 중 가장 완성된 형태의 클라우드 서비스이다.
    • ex - Google App , Dropbox

하이퍼 바이저

가상 머신(VM)을 생성하고 실행하는 프로세스

하이퍼 바이저를 사용함으로서 얻는 이점은 하드웨어 독립성, 효율성, 확장성, 이동성이 있음

하이퍼바이저는 가상화 또는 하드웨어와 소프트웨어 분리를 지원하는 기반 기술

-> 여러 업무 부서가 단일 서버에서 여러 가상 머신을 사용하여 서로 다른 워크로드를 별도로 실행 가능할 수 있음 (=업무 세분화)

하이퍼 바이저에게는 2가지의 유형이 존재

  1. 베어 메탈

      2. 호스트의 하드웨어에서 직접 구동되어 게스트 운영 체제를 관리

      ex - KVM, Hyper-V

  2. 호스팅

      3. 기존의 운영 체제에서 소프트웨어 레이어 또는 에플리케이션으로서 구동됨

      ex - VMware Workstation, VirtualBox

rm -rf와 위험성

UNIX 계열의 운영체제에서 모든 파일 (자기 자신을 포함한)을 전부 삭제하는 명령어

터미널에서 디렉토리를 삭제할 때, rm - rf ./* 를 입력해버려 전부 포맷해버릴 수 있음

grep

특정 파일에서 지정한 문자열이나 정규 표현식을 포함한 행을 출력해주는 리눅스 명령어

grep '찾으려는 문자열' 파일명1 파일명2.확장자명

이런 식으로 사용

정규 표현식을 사용하여 찾을 수도 있음

  • grep '^[ab]' 파일명
    • 특정 파일에서 문자열이 포함된 행을 찾음
  • grep '[a-b]' 파일명
    • 특정 파일에서 a,b,c로 시작하는 단어를 모두 찾음
  • grep 'a...z' 파일명
    • 특정 파일에서 a로 시작하고 z로 끝나는 5자리 단어를 찾음
  • grep 'a*' 파일명
    • 특정 파일에서 a로 시작하는 모든 단어를 찾음

파일 사용자와 그룹

리눅스 파일 사용자/사용 그룹 변경에 사용되는 명령어들

    chown

    파일, 디렉토리의 권한을 변경(소유권을 변경)하는 역할

  • chown user1 html1.txt

    • 소유자만 변경 , 이전 소유자 -> user1

  • chown :htmlgroup html1.txt

    • 그룹만 변경 , 이전 그룹 -> htmlgroup

  • chown user1:htmlgroup html1.txt

    • 그룹,소유자 둘 다 변경 , 이전 소유자:그룹 -> user1:htmlgroup

    chmod

    파일, 디렉토리의 권한을 변경(값을 변경)하는 역할

    -rwx/rwx/rwx , 파일 소유 사용자 권한 / 파일 소유 그룹 권한 / 그 외 사용자 권한

  • chmod 664 html1.txt

    • html1.txt 파일을 , 파일 소유 사용자 권한 (6) = rw- , 파일 소유 그룹 권한 (6) = rw- , 그 외 사용자 권한 (4) = -w-

      따라서 'ls -l' 명령을 통해 알아낼 수 있는 파일의 권한은 -rw-rw--w- 이다

netstat

네트워크 연결 상태, 라우팅 테이블, 인터페이스 상태 등을 보여주는 명령어

netstat [옵션] [포트번호 or 서비스 명]

    옵션 종류
  • -a : 현재 다른 PC와 연결되어 있거나 대기중인 모든 포트 번호를 확인
  • -r : 라우팅 테이블 확인 및 커넥션되어 있는 포트번호를 확인
  • -n : 현재 다른 PC와 연결되어 있는 포트번호를 확인
  • -s : IP,ICMP, UDP 프로토콜별의 상태 확인
  • -c : 1초 단위로 보여줌

iptables

리눅스상에서 방화벽을 설정하는 도구로서 패킷을 제어하는데 사용

iptables의 제어 규칙을 정의하는 기본 단위는 Chain임

Chain은 패킷이 조작될 상태를 지정하며, iptables에 내장된 기본 Chain은 3가지가 존재

  • Chain INPUT : 서버로 들어오는 기본 정책

    • Linux Server를 목적지로 삼는 모든 패킷이 통과하는 Chain

  • Chain FORWARD : 서버에서 forwarding 기본 정책

    • Linux Server에서 생성돼 외부로 보내지는 모든 패킷이 통과하는 Chain

  • Chain OUTPUT : 서버에서 나가는 기본 정책

    • Linux Server를 목적지로 삼지 않는 패킷이 통과하는 Chain

ufw

우분투 방화벽 (Ubuntu FireWall)

ufw는 우분투의 기본적인 방화벽이다

iptables를 쉽게 설정할수 있도록 하지만, 이는 간단한 방화벽 구성에만 적합하며 보다 복잡한 구성에는 iptables을 직접 사용해야 함

  • ufw 설치

    • sudo apt-get install ufw

  • ufw 상태 관리 명령어

    • # ufw 활성화 / 비활성화

      sudo ufw enable / disable

    • # 상태 확인

      sudo ufw status verbose

  • ufw 기본 룰

    • # 기본 룰 확인

      sudo ufw show raw

    • # 기본 정책 차단 / 허용

      sudo ufw default dney / allow
