1.1 RDP정의와 공격

RDP (Remote Desktop Protocol)

원격 데스크톱 프로토콜(RDP)은 Microsoft에서 개발한 사유 프로토콜이며, 다른 컴퓨터에 GUI를 제공하는 프로토콜이다.

기본적으로 TCP 포트 3389를 사용한다.

원격으로 컴퓨터를 제어 가능하다는 점이 해커들의 공격에 있어 취약점을 가지고, 기본적으로 사용되는 포트값을 통해 쉽게 공격받는다는 단점을 가진다.

1.2 타입스크립트와 자바스크립트

자바 스크립트 (Java Script)

웹 프로그래밍에 사용되는 프로그래밍 언어중 하나로 스크립팅 언어라고도 한다.

동적인 정보를 보여주는것에 특화되어 있다. 예를 들자면, 주기적으로 갱신되거나, 상호작용이 가능한 경우 자바 스크립트로 구현되어 있을 가능성이 있다.

웹을 구성할 때 html, css, 자바 스크립트 총 3개로 구성할 떄, html로는 웹의 내용을 작성하고, css로는 웹을 디자인하며, 자바 스크립트로는 웹의 동작을 구현 할 수 있음

객체 기반의 스크립트 언어로, 타입을 명시할 필요가 없는 인터프리터 언어임 그리고 객체 지향형 프로그래밍과 함수형 프로그래밍을 모두 표현 가능한 언어임

자바와 자바스크립트 각 언어는 직접적인 관련은 없는 엄연히 다른 언어이며 C언어 기반이라 문법이 조금 비슷한 점만 있을 뿐이다

타입 스크립트 (Type Script)

타입 스크립트는 자바 스크립트에 타입을 부여한 언어 즉, 자바 스크립트의 확장된 언어이다.

기존의 자바 스크립트에서 기능을 추가한 것 이라고 생각하면 된다.

    TypeScript의 특징
  • 정적 타입 - 가독성 향상, 에러 메시지의 퀄리티가 높아 디버깅이 쉬움
  • 실행 속도 - 동적 타입(자바 스크립트)은 타입을 결정 -> 적용의 시간이 걸리는 반면,
    정적 타입(타입 스크립트)은 이러한 과정이 없기 때문에 실행 속도가 빠르다는 장점을 가짐

1.3 CI/CD

CI/CD (Continuous Integration / Continuous Delivery or Deployment)

애플리케이션 개발 단계를 자동화하여, 애플리케이션을 더욱 짧은 주기로 고객에게 제공하는 방법

CI - 지속적인 통합

코드 변경 사항이 정기적으로 빌드 및 테스트 되어 공유 repository에 통합하는 것을 의미

CD - 지속적인 서비스 제공 , 지속적인 배포

개발자의 변경 사항이 고객의 프로덕션 환경까지 통합되는 것을 의미

-> 서비스의 개발팀과 비즈니스팀 간의 커뮤니케이션 부족 문제를 해결, 배포에 이르기까지 노력을 최소한 단축

1.4 디지털 포렌식

디지털 포렌식

디지털 증거물을 분석하여, 수사에 활용하는 과학 수사 기법을 말함

해킹과 비교하자면, 해킹은 불법적인 루트를 통해 정보를 추출하여 악이용하는 것이고, 디지털 포렌식은 합법적인 경로로 접근하여 정보를 추출하는 방식이다.

디지털 포렌식의 종류로는 컴퓨터 포렌식, 임베디드 포렌식, 네트워크 포렌식이 있다.

  • 컴퓨터 포렌식 - Windows나 Unix와 같은 운영 체제를 탑재한 범용 컴퓨터를 대상으로 하는 디지털 포렌식
  • 임베디드 포렌식 - 핸드폰과 같은 모바일 기기나 디지털 카메라와 같은 다양한 디바이스에 대한 디지털 포렌식
  • 네트워크 포렌식 - 통신 디바이스에서 네트워크 정보, 사용자 로그 등의 정보를 수집 및 분석하는 포렌식

1.5 UI/UX

UI/UX (User Interface / User Experience)

UI - 사용자 인터페이스

사용자가 사용하는 서비스나 제품의 화면 안에 모든 것을 의미함

UX - 사용자 경험

사용자가 어떤 시스템, 서비스, 제품 등을 직, 간접적으로 이용하면서 느끼는 총체적 경험

1.6 리액트

리액트 (React)

오픈 소스 자바스크립트 라이브러리의 하나로서 UI를 만들기 위해 사용됨

싱글 페이지 애플리케이션이나 모바일 애플리케이션 개발이 사용될 수 있음

    리액트의 특징
  • 가상 DOM을 사용함 : 가상 DOM을 이용하여 연산을 최소화 -> 성능 최적화
  • 단방향 데이터 바인딩 : 양방향 데이터 바인딩에 비해 Watcher를 1개만 사용하여, 성능 최적화
  • JSX라는 문법을 가짐 : Java Script + Html
  • 선언형 프로그래밍 방식을 사용
  • 컴포넌트 기반 : 작고 고립된 코드를 사용하여 코드 유지보수 용이

1.7 스벨트

스벨트 (Svelte)

2016년 출시한 오픈 소스 프론트엔드 웹 프레임 워크이다.

가장 가볍고 빠른 앱을 지향하여, 최소한의 코드를 사용함

    스벨트의 특징
  • 실제 DOM을 사용함 : 가상 DOM의 조절 프로세스를 사용 X -> 성능 최적화
  • 간소화된 소스코드 : React에 비해 적은 양의 소스코드로도 구현해 낼 수 있음

1.8 리덕스

리덕스 (Redux)

자바스크립트의 상태관리 라이브러리이다.

복잡하게 얽혀있는 여러가지 컴포넌트들에서 데이터를 쉽게 꺼내다 쓸 수 있게 해줌

위 사진처럼 중앙 집권 체제로 데이터를 운용하는 방식임 (주로 리액트에서 볼 수 있는 컴포넌트 구성)

리덕스에는 미들웨어(Middleware)라는 개념이 존재함 (액션 객체가 처리되기 전에 우리가 원하는 작업을 수행 할 수 있음)

1.9 플러터

플러터 (Flutter)

구글에서 2017년에 출시한 크로스 플랫폼 GUI 앱 프레임워크이다.

사용되는 언어는 구글에서 제창된 Dart를 사용함

플러터에 기본적으로 제공하는 UI들이 있어 쉽다는 장점이 있으며, 웹 개발과 개발 방식이 같고 자바스크립트 문법과 유사한 점이 많다.

1.10 REST-API

REST(Representational Strate Transfer) - API

REST를 기반으로 만들어진 API이다.

    REST-API의 구성
  • 자원 (Resource) - HTTP URI
  • 행위 (Verb) - HTTP Method
  • 표현 (Representations) - HTTP Message Pay Load
    REST의 특징
  • 서버-클라이언트 구조 (Server-Client)
  • 무상태 (Stateless)
  • 캐시 처리 가능 (Cacheable)
  • 계층화 (Layered System)
  • 인터페이스 일관성 (Uniform Interface)

1.11 Graph-QL

Graph-QL은 애플리케이션 프로그래밍 인터페이스(API)를 위한 퀴리 언어이다.

Graph-QL은 클라이언트에서 자기에 필요한 데이터만을 쿼리할 수 있도록 하여 문제를 매우 깔끔하고 직관적이게 해결함

Graph-QL은 클라이언트에서 사용할 쿼리 언어를 정의함 -> 클라이언트가 필요한 데이터에 대한 Query를 GraphQL에 넘김 -> GraphQL은 Query를 해석해 서버에서 필요한 데이터를 가져오고, 클라이언트에 해당 Query에 대한 데이터를 반환

1.12 Node.js

Node.js는 확장성 있는 네트워크 애플리케이션(Server-side에 특화) 개발에 사용되는 소프트웨어 플랫폼이다.

사용 되는 언어는 자바스크립트이고, 내장 HTTP서버 라이브러리를 포함하고 있어 웹 서버에서 별도의 소프트웨어 없이 동작하는 것이 가능함

Node.js의 특징으로는

    단일쓰레드 이벤트 루프 기반 비동기방식 (Non-Blocking I/O)
  • 하나의 쓰레드가 request를 받으면 바로 다음 처리에 요청을 보내놓고 다른 작업을 처리하다가 먼저 요청한 작업이 끝나면 이벤트를 받아서 응답을 보냄
  • 동시 request가 오더라도 처리가 완료될때까지 기다리지 않아도 되기 때문에 서버 부하가 적음

장점으로는 자바 스크립트를 동일하게 사용하고, 개발이 빠르고 쉬우며, 서버가 매우 가볍게 돌아감

단점으로는 이벤트 기반 비동기방식이라 서버단 로직이 복잡한 경우 수정이 어려울 수 있고, 하나의 작업 자체가 많이 걸리는 웹 서비스에 적합하지 않음

1.13 express-JS

express-JS는 Node.js를 위한 빠르고 개방적인 간결한 웹 프레임워크이다.

-> Node.js를 사용하여 쉽게 서버를 구성할 수 있게 만든 클래스와 라이브러리의 집합체

Express는 프레임워크이므로 웹 애플리케이션을 만들기 위한 각종 라이브러리와 미들웨어 등이 내장돼 있어 개발하기 편하고, 수많은 개발자들에게 개발 규칙을 강제하여 코드 및 구조의 통일성을 향상시킴

1.14 Nest-JS

Nest-JS는 Node.js 위에서 동작하는 타입 스크립트용 오픈 소스 백엔드 웹 프레임워크이다.

express-js는 매우 경량화된 프레임워크라서 규모가 큰 프로젝트에서는 구현해야 할 기능이 많아 불편함 -> 이러한 불편함을 해소하기 위해 등장한 프레임워크가 Nest-JS

기업용으로 사용 될 정도로 다양한 기능을 내장하고 있으며, 플러그인을 통해 확장도 가능함

객체 지향 프로그래밍, 의존성 주입, 과점 지향 프로그래밍등의 백앤드 개발에 최적화된 특징을 가지고 있음

1.15 Next-JS

Nest-JS는 React로 만드는 Server-Side Rendering 프레임워크이다.

서버사이드 렌더링을 함으로서 클라이언트 렌더링에 비해 사용자가 기다려야할 시간을 줄임

구글 검색엔진에 페이지가 노출되도록 함 -> 수익성 증가

1.16 Spring

Java/Kotlin 기반의 웹 프레임워크이다.

    Spring의 특징
  • POJO (Plain Old Java Object) 방식 - 별도의 프레임워크 없이 Java EE를 사용할 떄와 비교하면 특정 인터페이스를 구현,상속 받을 필요가 없어 기존 라이브러리를 지원하기 용이하고 객체가 가벼움
  • 관점 지향 프로그래밍 (Aspect Oriented Programming, AOP) - 추상/부모/클래스나 인터페이스로 관리된다는 게 아니라 모듈을 관리해주는 모듈을 상하/인터페이스 관계없이 따로 마련한다는 개념
  • 의존성 주입 (Dependency Injection, DI) - 프로그래밍에서 구성요소 간의 의존 관계가 소스코드 내부가 아닌 외부에서 설정을 통해 정의되는 방식으로, 코드 재사용율을 높여 모듈간의 결합도를 낮춰줄 수 있음

1.17 Electron

Github에서 유지보수하고 있는 GUI 프레임워크이다.

크로뮴과 Node.js에 기초하여, HTML, CSS, JavaScript를 이용하여 웹 앱을 제작하듯 데스크톱용 앱을 제작할 수 있다. 대표적인 예시로 Discord

배우기 쉽고, 방대한 자료가 존재하며, Node.js를 사용하여 JavaScript의 모든 기능을 사용할 수 있음

그러나 보안에 취약하며, 앱이 많이 무거워 resource를 많이 잡아먹음
